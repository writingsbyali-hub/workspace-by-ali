## What Goes in a Session Folder?

- **Session Handoffs** - Summary of work done, next steps
- **Current State Snapshots** - What's working, what's broken
- **Testing Results** - Browser tests, API tests, integration tests
- **Decision Logs** - Why we chose X over Y
- **Quick Start Guides** - How to get up and running after this session
- **Refactoring Trackers** - Migration progress

## Archive Policy

Session folders older than 2 weeks should be considered historical reference. For the latest current state, always check:
- [docs/MASTER_TASKLIST.md](../MASTER_TASKLIST.md) - Current priorities
- The most recent session folder
- [docs/architecture/](../architecture/) - Up-to-date architecture docs
